## Executive Summary

WSOPTV OTT 플랫폼의 **20개국 글로벌 런칭**을 위한 법규 준수 전략을 정의합니다. 특히 **칩 자동 생성(Cross-Promotion)**, **온라인 도박 규제**, **콘텐츠 스트리밍 규제**에 대한 국가별 분석과 대응 방안을 제시합니다.

### 주요 법규 리스크

| 리스크 영역 | 심각도 | 영향 국가 |
|------------|:------:|----------|
| 칩 자동 생성 (STRAT-0003 Flow 2) | **Critical** | 미국, 호주, 독일 등 |
| 도박 콘텐츠 스트리밍 | High | 한국, 일본, 중국 |
| VPN 우회 접속 | High | 전체 |
| 결제 규제 | Medium | 유럽 (PSD2) |

## 2. Critical Issue: 칩 자동 생성 규제

### 2.1 문제 정의

**STRAT-0003 Flow 2** (WSOPTV → GG POKER 칩 자동 생성):
- WSOPTV $10 구독 → GG POKER $10 칩 자동 생성
- WSOPTV $50 구독 → GG POKER $50 칩 자동 생성

**법적 쟁점**:
1. 칩 = 금전적 가치 → 도박 규제 적용 가능
2. "자동 생성" = 도박 유인 행위로 해석 가능
3. 미성년자 보호 규정 위반 가능성

### 2.2 국가별 분석

#### 미국 (주별 상이)

| 주 | GG POKER 합법 | 칩 자동 생성 | 대응 |
|---|:------------:|:----------:|------|
| 뉴저지 (NJ) | ✓ | ⚠️ 승인 필요 | DGE 협의 |
| 펜실베니아 (PA) | ✓ | ⚠️ 승인 필요 | PGCB 협의 |
| 미시간 (MI) | ✓ | ⚠️ 승인 필요 | MGCB 협의 |
| 기타 주 | ❌ | ❌ | 비활성화 |

**권장안**: 합법 주에서만 칩 자동 생성, IP 기반 지역 제한

#### 영국 (UKGC)

| 항목 | 상태 | 비고 |
|------|:----:|------|
| GG POKER 라이선스 | ✓ 보유 | GB-000-046540-R-327234-002 |
| 칩 자동 생성 | ⚠️ 조건부 | "bonus" 규정 적용 |
| 광고 제한 | ⚠️ 주의 | 18세 미만 타겟팅 금지 |

**권장안**: UKGC "bonus" 가이드라인 준수, 책임 도박 문구 표시

#### 호주 (ACMA)

| 항목 | 상태 |
|------|:----:|
| 온라인 포커 | ❌ 금지 (Interactive Gambling Act 2001) |
| 칩 자동 생성 | ❌ 금지 |
| VPN 우회 | ❌ 감지 시 차단 의무 |

**권장안**: 호주에서 칩 자동 생성 완전 비활성화, 순수 시청 서비스만 제공

#### 독일 (GlüStV 2021)

| 항목 | 상태 | 비고 |
|------|:----:|------|
| 온라인 포커 | ⚠️ 조건부 | 연방 라이선스 필요 |
| 칩 자동 생성 | ⚠️ 검토 중 | 보너스 규제 적용 가능 |
| 입금 한도 | €1,000/월 | 칩 생성에도 적용 가능 |

**권장안**: GG POKER 독일 라이선스 확인 후 제한적 운영

#### 한국/일본

| 국가 | 온라인 도박 | 칩 자동 생성 | 콘텐츠 시청 |
|------|:----------:|:----------:|:----------:|
| 한국 | ❌ 금지 | ❌ 금지 | ⚠️ 주의 |
| 일본 | ❌ 금지 | ❌ 금지 | ⚠️ 주의 |

**권장안**: 칩 자동 생성 완전 비활성화, 순수 OTT 서비스만 제공

### 2.3 칩 자동 생성 대안 (3가지)

#### 대안 A: 국가별 선택적 활성화

```
WSOPTV 구독 ─────────────────────────────────────────
                │
     ┌──────────┴──────────┐
     │                     │
칩 자동 생성 허용 국가    칩 자동 생성 금지 국가
(영국, 미국 합법 주)     (호주, 한국, 일본 등)
     │                     │
     ▼                     ▼
칩 자동 생성           순수 구독 혜택
                      (아카이브 추가 제공 등)
```

**장점**: 규제 준수, 최대 시장 커버리지
**단점**: 국가별 UX 차이, 개발 복잡도

#### 대안 B: "바우처" 모델

- 칩 직접 생성 대신 **GG POKER 바우처 코드** 발급
- 사용자가 직접 GG POKER에서 바우처 입력
- 규제 회피 가능 (사용자 주도적 행위)

**장점**: 법적 리스크 감소
**단점**: 전환율 하락 예상 (수동 절차)

#### 대안 C: 크레딧 시스템

- 현금 가치 칩 대신 **플레이 머니(크레딧)** 제공
- 리얼 머니 전환 불가
- 규제 대상 회피

**장점**: 전 세계 일관된 UX
**단점**: GG POKER 전환 유인 약화

#### 권장안

| 국가 유형 | 권장 대안 |
|---------|---------|
| 완전 합법 (영국 등) | 칩 자동 생성 유지 |
| 조건부 합법 (미국 일부 주, 독일) | 대안 A (선택적 활성화) |
| 금지 (호주, 한국, 일본) | 대안 C (크레딧) 또는 비활성화 |

## 4. VPN 우회 대응

### 4.1 PRD-0002 NFR-2 요구사항

| 요구사항 | 목표치 |
|---------|:-----:|
| VPN 감지 정확도 | 80-90% |
| 국가별 블랙아웃 | 지원 |

### 4.2 VPN 감지 기술

| 기술 | 정확도 | 비용 | 권장 |
|------|:-----:|:----:|:----:|
| IP 데이터베이스 (MaxMind) | 70% | 낮음 | ✓ |
| DNS Leak 탐지 | 60% | 낮음 | ✓ |
| WebRTC Leak 탐지 | 75% | 낮음 | ✓ |
| 타임존 불일치 | 65% | 낮음 | ✓ |
| **복합 탐지** | **85%** | 중간 | ✓ |
| 심층 패킷 분석 (DPI) | 95% | 높음 | - |

**권장 구현**:
1. MaxMind GeoIP2 + VPN 데이터베이스
2. WebRTC/DNS Leak 탐지 (클라이언트)
3. 타임존 및 언어 설정 검사
4. 이상 행동 패턴 탐지

### 4.3 VPN 감지 시 대응

| 위험 수준 | 대응 |
|---------|------|
| 낮음 (타임존 불일치만) | 경고 표시, 계속 허용 |
| 중간 (VPN IP 탐지) | 결제/칩 생성 차단, 시청 허용 |
| 높음 (다중 지표 일치) | 계정 일시 정지, 인증 요청 |

| 기록 보관 | 거래 기록 5년 보관 | P0 |

### 5.2 KYC (Know Your Customer)

| 국가 | KYC 수준 | 요구 서류 |
|------|:-------:|---------|
| 영국 | 엄격 | 신분증, 주소 증명 |
| 독일 | 엄격 | 신분증, 주소 증명 |
