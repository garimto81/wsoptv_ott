## 이슈 1: 멀티 뷰 vs 멀티 스크린 개념 정리

### 1.1 용어 정의

| 용어 | 정의 | 예시 서비스 | 핵심 특징 |
|------|------|------------|----------|
| **Multi-View** | 여러 개의 **다른 경기/테이블**을 동시 시청 | NBA TV | 4분할 화면에 각각 다른 경기 |
| **Multi-Screen** | 하나의 경기/테이블에서 **다른 앵글/카메라**를 동시 시청 | F1 TV Driver Cam | 같은 경기의 다른 시점 |

### 1.2 현재 문서 상태

#### Michael Note 원안 (Multi-Screen 개념)

```
Multi-view 영상: 메인화면이 중앙에 있고,
각 유저들의 얼굴을 잡고 있는 화면이 옆에 또 있는 방식 (아이돌 직캠 카메라)
```

**분석**: Michael의 원안은 용어는 "Multi-view"를 사용했으나, 실제 설명은 **Multi-Screen** (동일 테이블 내 플레이어별 직캠) 개념입니다.

#### NBA TV 참조 (Multi-View 개념)

PRD-0006에서 NBA TV 스타일의 **4분할 Multiview**를 채택:
- 여러 **다른 대회/테이블**을 동시에 시청
- "Add Table from Tournament Strip"

**분석**: NBA TV 매핑은 순수 **Multi-View** (여러 경기 동시 시청) 개념입니다.

#### 현재 PRD-0006 설계

현재 문서는 **두 가지를 모두 포함**:

| 옵션 | 개념 | 설명 |
|------|------|------|
| **옵션 A: 4분할 그리드** | Multi-View | NBA TV 스타일, 여러 테이블 동시 시청 |
| **옵션 B: 3계층 동적** | Multi-Screen | PGM → 피처 테이블 → PlayerCAM |

### 1.3 3계층 구조에서의 개념 매핑

```
┌──────────────────────────────────────────────────────────────┐
│  Layer 1: PGM (메인 PD 연출)                                   │
│           └── 전체 테이블 화면, PD가 컷팅/연출한 방송용 화면        │
├──────────────────────────────────────────────────────────────┤
│  Layer 2: Feature Tables (Multi-View)                        │
│           └── 여러 개의 다른 테이블 목록 (Main Event, High Roller 등)│
├──────────────────────────────────────────────────────────────┤
│  Layer 3: PlayerCAM (Multi-Screen)                           │
│           └── 선택한 테이블 내 각 플레이어별 직캠                  │
└──────────────────────────────────────────────────────────────┘
```

### 1.4 권장 용어 정리

| 기능 | 정확한 용어 | WSOPTV 명칭 | 설명 |
|------|------------|------------|------|
| 여러 테이블 동시 시청 | Multi-View | **Multiview** | NBA 스타일 4분할 |
| 같은 테이블 내 다중 앵글 | Multi-Screen | **PlayerCAM** | 플레이어별 직캠 |
| 통합 기능 | Hybrid | **3계층 Multi-view** | Layer 2(Multi-View) + Layer 3(Multi-Screen) |

## 이슈 2: 플레이어 직캠 소스 송출 경로

### 2.1 현재 프로덕션 워크플로우

```
┌─────────────────────────────────────────────────────────────────────┐
│                        현재 프로덕션 흐름                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  [라스베가스 현장]                                                     │
│       │                                                              │
│       ├── Feature Table 촬영 (메인 카메라)                            │
│       │                                                              │
│       └── 가편 작업 (현장 PD)                                         │
│               │                                                      │
│               ▼                                                      │
│  ──────── 라스베가스 → 한국 송출 ────────                              │
│               │                                                      │
│               ▼                                                      │
│  [한국 스튜디오]                                                       │
│       │                                                              │
│       └── 최종 PGM 종편 작업                                          │
│               │                                                      │
│               ▼                                                      │
│       ┌──────────────────┐                                           │
│       │   최종 PGM 출력   │ ← 현재 WSOPTV가 수신하는 소스               │
│       └──────────────────┘                                           │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 PlayerCAM 추가 시 필요한 워크플로우 변경

#### 현재 상황의 문제점

| 문제 | 설명 |
|------|------|
| **단일 소스** | 현재 한국으로 송출되는 것은 최종 편집된 PGM 1개 스트림 |
| **PlayerCAM 부재** | 개별 플레이어 직캠 피드가 한국까지 전달되지 않음 |
| **지연 발생** | 현장 가편 → 한국 종편 과정에서 30분+ 지연 |

#### PlayerCAM 송출을 위한 옵션

##### 옵션 A: 현장 직접 송출 (권장)

```
┌─────────────────────────────────────────────────────────────────────┐
│                        옵션 A: 현장 직접 송출                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  [라스베가스 현장]                                                     │
│       │                                                              │
│       ├── Feature Table 촬영                                         │
│       │       └── 메인 카메라 (PGM용)                                 │
│       │       └── PlayerCAM 카메라 (3-10대) ◀── 새로운 장비/인력       │
│       │                                                              │
│       └── 현장 스위처/인코더                                           │
│               │                                                      │
│               ├── PGM 피드 ──────────────┐                           │
│               │                          │                           │
│               └── PlayerCAM 피드들 ──────┼── CDN 직접 업로드          │
│                   (3-10개 스트림)        │                           │
│                                          ▼                           │
│                            ┌──────────────────────┐                  │
│                            │  OVP/CDN (클라우드)   │                  │
│                            │  - 동기화 처리        │                  │
│                            │  - 클라이언트 배포    │                  │
│                            └──────────────────────┘                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**장점**:
- 최소 지연 (현장 → CDN 직접)
- 한국 스튜디오 의존성 제거

**단점**:
- 현장 장비/인력 추가 필요
- 현장 네트워크 인프라 강화 필요

##### 옵션 B: 한국 스튜디오 경유

```
┌─────────────────────────────────────────────────────────────────────┐
│                        옵션 B: 한국 경유 송출                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  [라스베가스 현장]                                                     │
│       │                                                              │
│       └── 모든 카메라 피드 (PGM + PlayerCAM) ─────────────────────    │
│                                                                      │
│  ──────── 라스베가스 → 한국 다중 스트림 송출 ────────                   │
│                                                                      │
│  [한국 스튜디오]                                                       │
│       │                                                              │
│       ├── PGM 종편 작업                                               │
│       └── PlayerCAM 피드 패스스루 / 경미한 처리                         │
│               │                                                      │
│               ▼                                                      │
│       ┌──────────────────────────────────────────┐                   │
│       │ PGM (1개) + PlayerCAM (n개) → CDN 업로드  │                   │
│       └──────────────────────────────────────────┘                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**장점**:
- 기존 워크플로우 유지 가능
- 한국 스튜디오에서 품질 관리

**단점**:
- 송출 대역폭 대폭 증가 (스트림 수 × 대역폭)
- 추가 지연 발생
- 한국 스튜디오 장비 업그레이드 필요

##### 옵션 C: 하이브리드 (권장 대안)

```
┌─────────────────────────────────────────────────────────────────────┐
│                        옵션 C: 하이브리드                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  [라스베가스 현장]                                                     │
│       │                                                              │
│       ├── PGM용 메인 카메라 ─────────────────────────────────────────  │
│       │         │                                                    │
│       │         └──→ 한국 송출 (기존 워크플로우)                        │
│       │                                                              │
│       └── PlayerCAM 카메라들 ───────────────────────────────────────  │
│                 │                                                    │
│                 └──→ CDN 직접 업로드 (별도 경로)                       │
│                                                                      │
│  [한국 스튜디오]                                                       │
│       │                                                              │
│       └── PGM 종편 작업 (기존과 동일)                                  │
│                                                                      │
│  [OVP/CDN]                                                           │
│       │                                                              │
│       └── PGM + PlayerCAM 동기화 후 클라이언트 배포                    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**장점**:
- 기존 PGM 워크플로우 유지
- PlayerCAM은 별도 경로로 저지연 처리
- 점진적 도입 가능

**단점**:
- 동기화 복잡성 (두 경로의 지연 차이)
- 시스템 복잡도 증가

### 2.3 PlayerCAM 지원 범위

ADR-0001에서 정의된 대로, 모든 테이블이 PlayerCAM을 지원하지는 않습니다:

| 테이블 유형 | PlayerCAM 지원 | 이유 |
|------------|:-------------:|------|
| **High Roller 테이블** | ✅ 지원 | 고액 게임, 시청 니즈 높음 |
| **Main Event Final Table** | ✅ 지원 | 핵심 콘텐츠 |
| **Main Event 일반 테이블** | ⚠️ 부분 지원 | 인기 플레이어만 |
| **Side Event** | ❌ 미지원 | 촬영 대상 아님 |

### 2.4 기술적 요구사항

#### 필요 장비 (테이블당)

| 장비 | 수량 | 용도 |
|------|:---:|------|
| PlayerCAM 카메라 | 3-10대 | 플레이어별 클로즈업 |
| 로컬 스위처 | 1대 | 다중 피드 관리 |
| 인코더 | 1-2대 | 실시간 인코딩 |
| 네트워크 장비 | 1세트 | 고대역폭 업로드 |

#### 필요 대역폭 (예상)

| 스트림 유형 | 화질 | 비트레이트 | 테이블당 총합 |
|------------|------|----------|-------------|
| PGM | 1080p | 6-8 Mbps | 6-8 Mbps |
| PlayerCAM (각) | 720p | 2-4 Mbps | 6-40 Mbps (3-10캠) |
| **총합** | - | - | **12-48 Mbps/테이블** |

## 권장안

### Phase 1 (MVP)

1. **Multi-View (4분할)만 구현**
   - NBA TV 스타일 4분할 레이아웃
   - 여러 테이블 동시 시청
   - **PlayerCAM 미포함** (프로덕션 워크플로우 변경 불필요)

2. **기존 프로덕션 워크플로우 유지**
   - 현장 가편 → 한국 종편 → PGM 송출
   - 추가 인프라 투자 없이 시작 가능

### Phase 2

1. **PlayerCAM 추가 (옵션 C 하이브리드 권장)**
   - 현장 직접 CDN 업로드 경로 구축
   - Final Table부터 시작하여 점진적 확대

2. **3계층 동적 레이아웃 완성**
   - PGM → 피처 테이블 → PlayerCAM
