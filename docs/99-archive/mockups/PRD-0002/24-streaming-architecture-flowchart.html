Video Streaming Architecture v4
Camera to Viewer Flowchart | 26 Nodes, 7 Sections | YouTube + WSOPTV Split
Legend: Node Details (26 Nodes)
Sec Node Full Description Input / Output
A: On-Site Production
A Cameras HD/4K Broadcast Cameras (Main PGM, Player Cam, Table Cam, Audience) Scene -> SDI 1080p60 (1.5Gbps)
A Switcher Production Video Mixer/Switcher - camera source selection Multi SDI -> PGM + Clean Feed
A PokerGFX Poker Graphics Overlay Generator (POT, BOARD, Hole Cards, Win %) PGM ref + RFID -> GFX Layer (Alpha)
A Combiner Video Compositor - combines PGM with PokerGFX overlay PGM + GFX -> Combined SDI
A Encoder On-Site Live Encoder (H.264/H.265 compression) SDI 1.5Gbps -> RTMP/SRT 8-15Mbps
B: AWS Cloud
B MediaLive AWS MediaLive Ingest - receives stream from on-site encoder RTMP/SRT -> Cloud Stream
B Recording Cloud Archive Recording System (S3) - backup and VOD source Live Stream -> MP4/MKV segments
C: GG Production (Seoul)
C Ingest GG Production Ingest Point - registers to MAM system Cloud Stream -> MAM registered
C QC Quality Control - verifies video/audio quality, A/V sync MAM source -> QC PASS
C Cut Cut Editing - extract segments, remove dead air, mark ad points QC source -> EDL/XML timeline
C Hand Feature Table Hand Arrangement - select highlights, build storyline Timeline + metadata -> Sequence
C Subtitle Subtitle/Localization - 20 languages, poker terminology Sequence -> SRT/VTT (20 lang)
C Soft Soft Content Placement - sponsor logos, promos, SCTE-35 markers Subtitled -> Sponsor/Promo added
C Render Final Master Render - output final master video Complete seq -> ProRes 422 HQ
D: Transcoding & Packaging
D ABR Adaptive Bitrate Transcoder - multiple resolutions (1080p~240p) Master -> HLS/DASH ladder (0.3-8Mbps)
D DRM DRM Packaging - Widevine/FairPlay/PlayReady encryption Clear segments -> Encrypted + Manifest
D Origin Origin Server (S3 + Origin Shield) - serves to YouTube and CDN DRM packaged -> YouTube/CDN origin
E-1: YouTube Path (Free)
E1 YT RTMP YouTube RTMP Ingest - sends stream to YouTube Live Origin stream -> YouTube platform
E1 YT Live YouTube Live Platform - provides teaser to free viewers RTMP -> Viewer screen
E1 YT Viewer YouTube Viewer (Free) - target for WSOPTV conversion Live stream -> (END)
E-2: WSOPTV Path (Paid)
E2 CDN CDN Ingest (Multi-CDN) - pulls content from Origin Origin HLS/DASH -> CDN network
E2 Edge Edge Cache (Global POP) - caches near viewers (US, EU, APAC) CDN content -> HTTPS to viewer
E2 Viewer WSOPTV Viewer (Paid Subscriber) - full feature access Edge stream -> Section F
F: Client Playback (WSOPTV Only)
F Player WSOPTV Client Player (Web, iOS, Android, Samsung TV, LG TV) Edge stream -> DRM request
F DRM DRM Decryption (CDM) - acquires key, decrypts content Encrypted media -> Clear frames
F Render Video Rendering Engine - decode and display to screen Clear frames -> 1080p 60fps display
Protocol Reference
SDI Serial Digital Interface (uncompressed, 1.5Gbps)
RTMP Real-Time Messaging Protocol (streaming)
SRT Secure Reliable Transport (low-latency, FEC)
HLS HTTP Live Streaming (Apple)
DASH Dynamic Adaptive Streaming over HTTP
HTTPS TLS 1.3 encrypted transport
