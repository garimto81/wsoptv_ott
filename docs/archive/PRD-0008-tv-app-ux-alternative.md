## Executive Summary

**TV 앱(Samsung TV, LG TV)에서 Advanced Mode가 지원되지 않는 문제**에 대한 대안 UX 전략을 정의합니다. Plus+ ($50) 구독자가 TV에서도 프리미엄 가치를 경험할 수 있도록 4가지 대안을 제시하고, Phase별 권장안을 도출합니다.

### 문제 정의

| 플랫폼 | Advanced Mode | 제약 사유 |
|--------|:------------:|----------|
| Web | ✓ | - |
| iOS | ✓ | - |
| Android | ✓ | - |
| **Samsung TV** | **✗** | 리모컨 UX 제약 |
| **LG TV** | **✗** | 리모컨 UX 제약 |

**Plus+ 구독자 불만 시나리오**:
> "월 $50 내고 TV로 보는데 Multi-view도 StatsView도 안 되네요?"

---

## 1. Problem Statement

### 1.1 현재 상황

PRD-0006에서 정의된 Advanced Mode 기능:
- 4분할 Multi-view
- 3계층 동적 Multi-view (PGM → 테이블 → PlayerCAM)
- StatsView (HUD 오버레이)
- Key Hands 점프

이 기능들은 **터치 또는 마우스 인터랙션**을 전제로 설계됨.

### 1.2 TV 리모컨 제약

| 인터랙션 | Web/Mobile | TV 리모컨 |
|---------|:----------:|:---------:|
| 화면 탭/클릭 | ✓ | ✗ (방향키만) |
| 드래그 앤 드롭 | ✓ | ✗ |
| 동시 다중 선택 | ✓ | ✗ |
| 빠른 모드 전환 | ✓ | 느림 |
| HUD 토글 | ✓ | 복잡 |

### 1.3 비즈니스 영향

| 시나리오 | 영향 |
|---------|------|
| Plus+ 구독자 TV 시청 불만 | 이탈률 증가 |
| Plus+ 가치 인식 저하 | 전환율 저하 |
| 경쟁사 대비 열위 | 시장 점유율 손실 |

## 2. Proposed Solutions (4가지 대안)

### 2.1 옵션 A: 2분할 간소화 Multi-view

**개념**: 4분할을 2분할로 단순화, 리모컨으로 조작 가능

**레이아웃**:
```
┌─────────────────────────┬─────────────────────────┐
│                         │                         │
│     Main Stream         │     Secondary Stream    │
│     (PGM / Table)       │     (Table / PlayerCAM) │
│                         │                         │
│                         │                         │
└─────────────────────────┴─────────────────────────┘
        ↑                           ↑
    [Left/Right 키로 전환]    [Enter 키로 스왑]
```

**리모컨 조작**:

| 버튼 | 동작 |
|------|------|
| **Left/Right** | 좌/우 화면 포커스 이동 |
| **Enter** | 선택된 화면을 메인으로 스왑 |
| **Up/Down** | 스트림 목록 선택 |
| **Back** | 단일 화면 모드로 복귀 |

**장점**:
- 기존 Multi-view 컨셉 유지
- 리모컨으로 직관적 조작
- 개발 복잡도 낮음

**단점**:
- 4분할 대비 시청 옵션 감소
- 3계층 구조 표현 제한

**구현 난이도**: 중간
**권장 Phase**: Phase 1 (MVP)

---

### 2.2 옵션 B: Second Screen Companion

**개념**: TV는 메인 화면, 모바일로 Multi-view/StatsView 조작

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│                    TV (Main Display)                         │
│                                                              │
│          ┌─────────────────────────────────────┐             │
│          │                                     │             │
│          │       선택된 스트림 재생             │             │
│          │       (Full Screen)                 │             │
│          │                                     │             │
│          └─────────────────────────────────────┘             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
                            ▲
                            │ Sync
                            │
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│              Mobile (Companion Controller)                   │
│                                                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐     │
│  │  Table 1 │  │  Table 2 │  │ PlayerCAM│  │ StatsView│     │
│  │  (PGM)   │  │          │  │          │  │          │     │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘     │
│                                                              │
│  [탭하면 TV 화면 전환] [StatsView 모바일에서 표시]           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**작동 방식**:
1. TV 앱에서 "Companion 모드" 활성화
2. 모바일 앱에서 QR 코드 스캔 또는 코드 입력
3. 모바일에서 스트림 선택 → TV 화면 전환
4. StatsView는 모바일 화면에 표시

**장점**:
- Full Advanced Mode 경험 제공
- TV 대화면 + 모바일 조작 최적 조합
- 모바일 앱 가치 증가

**단점**:
- 모바일 기기 필수 (추가 기기 의존)
- 네트워크 동기화 지연 가능
- 개발 복잡도 높음

**구현 난이도**: 높음
**권장 Phase**: Phase 2

---

### 2.3 옵션 C: Voice Control Advanced Mode

**개념**: 음성 명령으로 Multi-view, StatsView 조작

**음성 명령 예시**:

| 명령 | 동작 |
|------|------|
| "멀티뷰 켜줘" | 2분할 Multi-view 활성화 |
| "테이블 2 보여줘" | 메인을 Table 2로 전환 |
| "플레이어캠 보여줘" | PlayerCAM으로 전환 |
| "스탯 보여줘" | StatsView 오버레이 표시 |
| "스탯 꺼줘" | StatsView 숨김 |
| "키핸드 보여줘" | Key Hands 목록 표시 |
| "다음 키핸드" | 다음 Key Hand로 점프 |

**구현**:
```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│                    TV (Samsung / LG)                         │
│                                                              │
│          ┌─────────────────────────────────────┐             │
│          │                                     │             │
│          │         Video Player                │             │
│          │                                     │             │
│          └─────────────────────────────────────┘             │
│                                                              │
│          🎤 "테이블 2 보여줘"                                  │
│                    │                                         │
│                    ▼                                         │
│          Voice Recognition (Bixby / ThinQ)                   │
│                    │                                         │
│                    ▼                                         │
│          WSOPTV Command Handler                              │
│                    │                                         │
│                    ▼                                         │
│          Stream Switch: Table 2                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**장점**:
- 핸즈프리 조작
- 미래 지향적 UX
- Samsung Bixby, LG ThinQ 활용

**단점**:
- 음성 인식 정확도 의존
- 소음 환경 문제
- 플랫폼별 API 상이
- 다국어 지원 복잡

**구현 난이도**: 높음
**권장 Phase**: Phase 3

---

### 2.4 옵션 D: Pre-rendered HUD Overlay

**개념**: StatsView HUD를 비디오에 사전 렌더링

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│   ┌───────────────────────────────────────────────────────┐ │
│   │                                                       │ │
│   │  [Phil Ivey]              Video Stream                │ │
│   │  VPIP: 28% PFR: 22%                                   │ │
│   │  3Bet: 12% AF: 3.2                                    │ │
│   │                                                       │ │
│   │                                       POT: $2.4M      │ │
│   │                                       Blinds: 50K/100K│ │
│   │                                                       │ │
│   └───────────────────────────────────────────────────────┘ │
│                                                              │
│   ↑ HUD가 비디오에 포함되어 송출됨 (별도 토글 불필요)         │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**작동 방식**:
1. 서버에서 HUD를 비디오에 오버레이 렌더링
2. Plus+ 구독자에게 "HUD 포함 스트림" 제공
3. 토글 불필요 (항상 표시 또는 별도 스트림 선택)

**스트림 옵션**:

| 스트림 | HUD | 대상 |
|--------|:---:|------|
| Standard | ✗ | Plus |
| With HUD | ✓ | Plus+ |

**장점**:
- TV 앱 개발 단순화
- 리모컨 조작 불필요
- 일관된 품질

**단점**:
- HUD ON/OFF 실시간 토글 불가
- 스트림 용량 2배 (HUD 포함/미포함)
- 유연성 감소

**구현 난이도**: 낮음
**권장 Phase**: Phase 1 (병행)

## 3. Recommendation (Phase별 권장안)

### 3.1 Phase별 구현 전략

```
Phase 1 (MVP - 2026 Q3)
│
├── 옵션 A: 2분할 간소화 Multi-view ★ 주요 구현
│   └── 리모컨으로 2분할 전환
│
└── 옵션 D: Pre-rendered HUD Overlay ★ 보조 구현
    └── Plus+ 전용 "Stats Stream" 제공

Phase 2 (2026 Q4)
│
└── 옵션 B: Second Screen Companion
    └── 모바일 컴패니언 앱

Phase 3 (2027)
│
└── 옵션 C: Voice Control
    └── Bixby / ThinQ 연동
```

### 3.2 권장 조합

| Phase | TV 기본 | Plus+ 추가 가치 |
|-------|--------|----------------|
| **Phase 1** | 단일 화면 + 2분할 | Pre-rendered HUD Stream |
| **Phase 2** | + Companion 지원 | Full Advanced (via Mobile) |
| **Phase 3** | + Voice Control | 음성 명령 Advanced Mode |

## 5. Technical Requirements

### 5.1 옵션 A: 2분할 구현

| 요구사항 | 스펙 |
|---------|------|
| 동시 스트림 | 2개 (메인 + 서브) |
| 레이아웃 | 50:50 분할 |
| 오디오 | 1개만 활성 (선택 가능) |
| 전환 시간 | < 500ms |
| 리모컨 매핑 | 방향키 + Enter |

**API 요구사항**:
```json
// GET /api/v1/content/{id}/streams
{
  "streams": [
    { "id": "pgm", "name": "PGM", "url": "..." },
    { "id": "table1", "name": "Table 1", "url": "..." },
    { "id": "table2", "name": "Table 2", "url": "..." },
    { "id": "cam_ivey", "name": "PlayerCAM - Ivey", "url": "..." }
  ]
}
```

### 5.2 옵션 D: Pre-rendered HUD

| 요구사항 | 스펙 |
|---------|------|
| 렌더링 | 서버 사이드 (OBS/FFmpeg) |
| HUD 데이터 | 실시간 업데이트 (5초 간격) |
| 스트림 분리 | Standard / With Stats |
| 추가 비용 | 인코딩 서버 + 스토리지 |

**스트림 선택 API**:
```json
// GET /api/v1/content/{id}/streams?include_stats=true
{
  "streams": [
    { "id": "standard", "name": "Standard", "has_hud": false },
    { "id": "with_stats", "name": "With Stats", "has_hud": true, "tier": "plus_plus" }
  ]
}
```
