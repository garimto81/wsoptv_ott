## 2. GG Production 피드백 요청

### 수신인
- GG Production 기술팀

### 제목
```
[WSOPTV] WSOPTV Station 추가를 위한 워크플로우 변경 비용 견적 요청
```

### 본문

안녕하세요,

WSOP TV OTT 플랫폼 구축과 관련하여 **WSOPTV Station** 추가에 따른 워크플로우 변경 비용 견적을 요청드립니다.

#### 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **작업명** | 현장 가편 → 서버 분리 전송 |
| **목적** | Feature Table PGM을 WSOPTV 서버로 실시간 추가 전송 |
| **참조 문서** | REPORT-2026-01-30, TECH-0001 |

#### 2. 현재 워크플로우

```
현장(Las Vegas) → LiveU Cloud → 서울(GG Production)
→ 핸드 분할 → 1-2hr 딜레이 → YouTube 방송
```

#### 3. 추가 워크플로우 옵션

다음 3가지 옵션에 대해 각각 비용 견적을 부탁드립니다:

##### 옵션 A: 현장 직접 송출
```
현장 Feature Table PGM → WSOPTV 서버 (직접 전송)
```

| 항목 | 확인 필요 사항 |
|------|---------------|
| 네트워크 대역폭 | 추가 필요한 대역폭 (Mbps) |
| 전송 프로토콜 | SRT/RTMP/HLS 중 권장 방식 |
| 장비 추가 | 현장 인코더/서버 추가 필요 여부 |
| 지연 시간 | 예상 지연 시간 (초) |
| 초기 구축 비용 | $ |
| 월간 운영 비용 | $ |

##### 옵션 B: 한국 경유
```
현장 → LiveU Cloud → 서울(GG Production) → WSOPTV 서버
```

| 항목 | 확인 필요 사항 |
|------|---------------|
| 서울 서버 증설 | 추가 서버/스토리지 필요 여부 |
| 네트워크 비용 | 서울 → WSOPTV 전송 비용 |
| 작업 인력 | 추가 모니터링/관리 인력 필요 여부 |
| 지연 시간 | 예상 지연 시간 (초) |
| 초기 구축 비용 | $ |
| 월간 운영 비용 | $ |

##### 옵션 C: 하이브리드
```
메인 PGM: 현장 → WSOPTV 서버 (직접)
Player Cam: 현장 → 서울 → WSOPTV 서버
```

| 항목 | 확인 필요 사항 |
|------|---------------|
| 분기 전송 설정 | LiveU Cloud 라우팅 설정 변경 필요 여부 |
| 동기화 이슈 | 지연 시간 차이로 인한 동기화 문제 |
| 초기 구축 비용 | $ |
| 월간 운영 비용 | $ |

다음 표를 작성하여 회신 부탁드립니다:

| 옵션 | 초기 비용 | 월간 비용 | 지연 시간 | 권장도 | 비고 |
|------|----------|----------|----------|:------:|------|

- 현재 LiveU Cloud 계약 조건 확인 필요

## 3. OTT 업체 (메가존) 피드백 요청

### 수신인
- 메가존 클라우드 OTT 솔루션팀

### 제목
```
[WSOPTV] 다중 소스 시간대 동기화 구현 비용 견적 요청
```

### 본문

안녕하세요,

WSOP TV OTT 플랫폼 구축과 관련하여 **다중 소스 시간대 동기화** 기능 구현 비용 견적을 요청드립니다.

#### 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **기능명** | 다중 소스 Frame-accurate Synchronization |
| **목적** | Multi-view/Player Cam 시간 동기화 |
| **소스 수** | 1개 메인 PGM + 8개 Player Cam (Feature Table당) |
| **참조** | PRD-0005 RFP v2.0, REPORT-2026-01-30 |

#### 2. 기술 요구사항

| 항목 | 요구사항 |
|------|---------|
| **동기화 방식** | LTC Timecode 기반 또는 동등한 방식 |
| **정확도** | Frame-accurate (1/30초 = 33ms 이내) |
| **소스 형식** | HLS/DASH 스트림 |
| **지연 허용** | 전체 파이프라인 30초 이내 |

#### 3. 확인 필요 항목

| # | 항목 | 세부 내용 |
|:-:|------|----------|
| 1 | **LTC Timecode 지원** | 기본 플랫폼에서 지원 여부 |
| 2 | **Frame-accurate Sync** | 개발 필요 여부 및 개발 비용 |
| 3 | **서버 인프라** | 추가 필요한 서버/스토리지 사양 |
| 4 | **CDN 비용** | 다중 소스 전송으로 인한 추가 CDN 비용 |
| 5 | **모니터링 도구** | 동기화 상태 모니터링 대시보드 제공 여부 |
| 6 | **버퍼링 영향** | 동기화 처리로 인한 추가 지연 시간 |
| 7 | **유지보수** | 월간 유지보수 및 모니터링 비용 |
| 8 | **개발 기간** | 기능 개발 및 테스트 소요 기간 |

#### 4. 회신 형식

다음 표를 작성하여 회신 부탁드립니다:
